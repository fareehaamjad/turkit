Overview

TurKit may be viewed as a Java/JavaScript API for running iterative tasks on Mechanical Turk. The primary feature of TurKit is the ability to safely reexecute TurKit programs without reexecuting costly side effecting functions on Mechanical Turk (like creating new HITs), while still allowing the programmer to write in a straight-line imperative manner—there is no need to unravel the program into a state machine.

  • For a more detailed overview, please see this paper.
